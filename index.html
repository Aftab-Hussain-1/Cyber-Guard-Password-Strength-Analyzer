Entropy, bit by bit

Estimated as length × log₂(character-set size). A longer alphabet and more characters both push this number up — and every added bit doubles the guesses an attacker needs.

⏱

Two attacker models

The offline model assumes a stolen hash and ten billion guesses a second on dedicated hardware. The throttled model assumes a rate-limited login form at roughly one hundred guesses a second — the gap between the two is the whole argument for rate limiting.

⚠

Beyond raw length

A long password can still fail instantly. This checker screens against a short list of common leaked strings and obvious keyboard runs, since those are cracked first regardless of character count.
